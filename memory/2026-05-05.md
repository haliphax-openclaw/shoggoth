## Issue #28 Investigation: Streaming output on Discord stops at character limit until turn is finished

### Problem Summary

When streaming responses on Discord, if the accumulated text exceeds the 2000-character limit, the streaming stops updating and the message remains stuck at the limit until the entire agent turn completes.

### Root Cause Analysis

#### Location of Issue

1. **`packages/daemon/src/presentation/turn-orchestrator.ts`** (lines 133-134)
   - `sliceDisplayText` function truncates accumulated text to `maxLen` (2000 chars)
   - This is called on **every streaming update** during the turn

2. **`packages/daemon/src/messaging/inbound-session-turn.ts`** (lines 134-142)
   - The streaming callback uses `sliceDisplayText` on accumulated text
   - When text exceeds 2000 chars, it gets truncated
   - Only the truncated text is pushed to the stream

3. **`packages/platform-discord/src/streaming.ts`** (lines 55-75)
   - The `setFullContent` method only handles **final** text at turn completion
   - It splits the final text into chunks but doesn't handle incremental updates properly during streaming

#### The Flow

1. Model generates text → `onModelTextDelta` callback fires with accumulated text
2. Accumulated text is passed through `sliceDisplayText` → truncated to 2000 chars
3. Truncated text is pushed to stream via `streamPusher.push(sliced)`
4. Discord message is updated with truncated text
5. As more text accumulates, it keeps getting truncated → message stays at ~2000 chars
6. At turn completion, `setFullContent(finalText)` is called
7. Final text is split into chunks and sent as separate messages
8. **But** the first message (the original streaming message) remains at 2000 chars until this point

### Expected Behavior

During streaming, when accumulated text exceeds 2000 characters:

1. The streaming message should be updated with the first 2000 characters
2. As more text accumulates, additional messages should be sent to accommodate the overflow
3. The streaming should continue to show progress, not get stuck

### Actual Behavior

During streaming, when accumulated text exceeds 2000 characters:

1. The streaming message gets truncated to 2000 characters
2. No additional messages are sent during streaming
3. The message remains stuck at 2000 characters until turn completion
4. At completion, the full text is split into chunks and sent as separate messages
5. **But** the user sees a static message for the duration of the turn

### Possible Solutions

1. **Incremental chunking during streaming**: Modify the streaming mechanism to send additional messages when the accumulated text exceeds the limit during streaming, not just at the end.

2. **Multi-message streaming**: Track which chunks have been sent during streaming and only send new chunks as they become available.

3. **Improved truncation display**: At least indicate to the user that more text is coming (e.g., append "..." or similar indicators).
